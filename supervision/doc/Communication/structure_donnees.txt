// Ce fichier décrit les structures de données et variables contenant les données nécessaire à l'application de supervision

Le fichier de configuration donne le nombre de robot et de postes du système. Ces valeurs sont stockées dans des variables globales :
- Le nombre de robots : int nb_robot;
- Le nombre de postes de la branche nord (103 - 105): int nb_poste_nord;  
- Le nombre de postes de la branche sud (102 - 104): int nb_poste_sud;  

Les autres variables globales de l'application sont : 
- Le numéro de la mission sélectionnée dans le tableau du sous-écran 2 : int numero_mission_select;
- Le numéro du robot associé à la mission selectionnée dans le tableau du sous-écran 2 : int num_robot_select;
- Le nombre de poste : int nb_poste;
- Le numéro du dernier poste de la branche nord (103 - 105) : int id_last_poste_nord;
- Le numéro du dernier poste de la branche sud (102 - 104) : int id_last_poste_sud;
- La taille en nombre de ligne du tableau dynamique affiché dans sous-écran 2 : int taille_tab_mission;
- Le nombre de mission actuellement dans la liste.

Le programme de supervision conserve dans des classes un certain nombre de données qu'il extrait des messages reçue par la liaison série, en provenance de la centrale. 

Les messages sont stockés dans une liste FIFO de chaines de caractères : StringList fifo_messages;
La liste pourra contenir au maximum 100 messages.
La taille maximale d'un message entrant est de 8 caractères de 8bits chacun.
-> Taille totale de la liste : 100 éléments de taille 64 bits.

----

Classe ROBOT:
    - Un robot possède un identifiant : int id_robot;
    - Un robot possède une position sur le circuit, donnée par l'id du dernier poste à l'avoir vu : int id_last_poste;
    - Un robot possède une vitesse : int vitesse;
    - Un robot possède un status : char status;
    - Un robot possède une situation, soit sur le circuit, soit garé à coté d'un poste : boolean gare;
    - Un robot possède une disponibilité : boolean dispo;

Classe POSTE:
    - Un poste possède un identifiant : int id_poste;
    - Un poste possède un indicateur de robot vu : boolean robot_vu;
    - Un poste possède un indicateur de robot garé : boolean robot_gare;
    
Classe MISSION:
    - Une mission a un poste de départ désigné par son id : int p_depart; 
    - Une mission a un poste d'arrivé désigné par son id : int p_arrive;
    - Une mission a un robot livreur attribué désigné par son id : int id_livreur;  
    - Une mission a un code livreur : int code_livreur;

----

L'application de supervision garde en mémoire l'ensemble des postes et robots existant dans le système tel que spécifié dans le fichier de configuration :
- Chaque robot du système est une instance de la classe ROBOT. Ils sont stockées dans un tableau de ROBOT à l'indice id_robot dont la taille est nb_robot : Robot[] listeRobot;
- Chaque poste du système est une instance de la classe POSTE. Ils sont stockées dans un tableau de POSTE à l'indice id_poste dont la taille est nb_poste : Poste[] listePoste;

Chaque mission reçus via un message est une instance de la classe MISSION. Elles sont stockés dans un tableau dynamique de MISSION : Mission[] listeMission;.
